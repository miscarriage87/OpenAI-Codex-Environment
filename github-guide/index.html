GitHub Guide

Was ist GitHub?

GitHub ist ein Hosting-Service für Git-Repositories.

Grundlegende Befehle

Mit git clone <URL> lädst du ein Repository herunter.
Ein Commit speichert deine Änderungen mit git commit -m "Nachricht".
Mit git push lädst du deine Commits zu GitHub hoch.
git pull holt neue Änderungen vom Remote-Repository.
Ein Branch ist ein paralleler Entwicklungszweig. git checkout -b mein-branch erstellt ihn.

Typischer Workflow

1️⃣

Repository klonen

2️⃣

Dateien bearbeiten und committen

3️⃣

Commits zu GitHub pushen

4️⃣

Neues vom Remote mit pull holen

5️⃣

Branches und Pull Requests verwenden

Wie Codex GitHub nutzt

  1. Repo klonen oder aktualisieren
  2. Neuen Branch anlegen und Dateien anpassen
  3. Tests ausführen und Ergebnisse sammeln
  4. Commit erstellen und Branch pushen
  5. Pull Request erzeugen und Zusammenfassung liefern
