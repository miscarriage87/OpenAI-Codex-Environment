GitHub leicht erklärt

Diese kurze Seite gibt einen Überblick über grundlegende Begriffe und die Arbeitsweise von Codex.

Was ist GitHub?

GitHub ist ein Hosting-Service für Git-Repositories. Git selbst ist ein verteiltes Versionskontrollsystem, mit dem du den Entwicklungsstand deiner Dateien über Commits festhältst.

Wichtige Befehle

  • clone: Erstellt eine lokale Kopie eines Git-Repositories von einem Server.
  • commit: Hält eine Änderung im Repository fest.
  • push: Überträgt lokale Commits zu einem entfernten Repository, z. B. auf GitHub.
  • pull: Holt neue Änderungen vom Remote-Repository und integriert sie in dein lokales.
  • branch: Ein unabhängiger Entwicklungsstrang. Mit git checkout -b neuer-branch erstellst du einen neuen Branch.

Typischer Ablauf

  1. Repository klonen: git clone <URL>
  2. Dateien ändern und über git add zur nächsten Commit vorbereiten.
  3. Mit git commit die Änderungen speichern.
  4. Mit git push die Commits zu GitHub hochladen.
  5. Um neue Änderungen anderer einzuholen, git pull verwenden.

Branches und Pull Requests

Branches erlauben parallele Entwicklung, ohne die Hauptversion zu stören. Änderungen eines Branches werden über einen Pull Request (PR) in den Hauptbranch integriert.

Wie Codex damit arbeitet

Wenn du Codex einen Task gibst, interagiert es mit einem Git-Repository etwa folgendermaßen:

  1. Klone das Repository (falls nicht schon lokal vorhanden).
  2. Erstelle oder verwende einen Branch, um die gewünschten Dateien zu bearbeiten.
  3. Führe mögliche Tests aus und sammle die Ausgabe.
  4. Commite die Änderungen mit einer passenden Nachricht.
  5. Pushe diesen Branch zu GitHub und erstelle dort (falls gewünscht) einen Pull Request.
  6. Codex gibt dir typischerweise eine Zusammenfassung der Schritte, inkl. evtl. Testergebnisse.
